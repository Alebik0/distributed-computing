# Manytask

Привет! Добро пожаловать в онбординг по среде выдачи и сдачи заданий Manytask для студента!

Менитаск был создан, чтобы с одной стороны автоматизировать проверку заданий на курсах, где необходимо писать код, а с другой стороны сразу познакомить обучающихся с популярными инструментами разработки: git, gitlab CI и другими языко-зависимыми тулзами.

** Важно! ** Для удобства знакомства с системой был создан отдельный "курс" под названием onboarding. Однако скорее всего, вам интересен какой-то конкретный курс. Если так, то в большинстве случаев достаточно заменить название курса на актуальное и предоставить кодовое слово (вам его сообщат на курсе).

Схема менитаска представлена на картинке ниже, он состоит из следующих блоков:
* [py.manytask.org](https://app.manytask.org)  
  Здесь вы можете видеть все доступные задачки, дедлайны и свои баллы. 
* [gitlab.manytask.org](https://gitlab.manytask.org/)  
  Наш инстанс gitlab где будет находиться ваш личный удаленный репозиторий, публичный репозиторий курса и проверяться задания. 
* [публичный репозиторий](https://gitlab.manytask.org/spbgu/distsys/public-2025-fall) 
  Репозиторий в который регулярно выкладываются новые задачки (или фиксы к старым)  
  В процессе настройки вы подключите его как upstream для обновления и получения новых задачек (будет ниже)  
* [ваш личный удаленный репозиторий]  
  Основная точка работы с курсом - решения задачек и их проверки. 
  Является форком от публичного репозитория.  
  Репозиторий вы получите **автоматически** в процессе регистрации, вам нужно будет его только скопировать в Локальный репозиторий. Однако обновления задачек в нем самостоятельно не будут появляться.
* [ваш личный локальный репозиторий]
  Работать с кодом удобнее на своем ноутбуке, поэтому при подготовке к сдаче заданий вам предстоит склонировать ваш удаленный репозиторий себе на ноутбук. Непосредственное написание решений будет происходить локально. Затем готовые решения будут заливаться в ваш удаленный репозиторий, где их проверят раннеры. 
* [gitlab runner](https://docs.gitlab.com/runner/)  
  Система запуска докер контейнеров где и будет запускаться код вашего решения.
  Раннеров ограниченное количество, поэтому на границе дедлайна раннеры будут перегружены, и время проверки возрастет. Так что настоятельно рекомендуется сдавать задачки заранее, пока дедлайн еще не близко ;)

## Пошаговая инструкция по подготовке к сдаче заданий

В этой инструкции описываются шаги настройки репозитория и курсонезависимой среды для сдачи заданий. Тут покрываются только общие для разных курсов вещи, скорее всего понадобится дополнительная настройка для решения задачек в каждом из курсов. Предполагается, что вы имеете базовые знания работы с командной строкой и git. Эта инструкция и основы работы с git покрываются на онбординговой лекции (перезаписывается каждый год, скорее всего вы знаете где ее искать).


1. Заходим на https://app.manytask.org/ и заполняем форму регистрации. Если вы ранее сдавали что-то через менитаск, то скорее всего этот шаг можно пропустить, попробовав кликнуть Login. Для онбординга вам нужно написать название курса `onboarding-2025-fall` и секретный код `RebaseCamp`, для других курсов код другой. Имя и фамилию нужно писать на русском так же, как вы записаны в образовательном учреждении, чтобы куратор курса мог в конце курса без проблем понять кому принадлежит аккаунт менитаска и проставить оценку за курс в ведомость. Некоторые курсы дополнительно имеют другие методы связи менитаска и учебного аккаунта, но на всякий случай имя следует указывать корректно. Ник можно выбирать произвольный на латинском языке, требования те же, что и для регистрации аккаунта в гитлабе (латиница без пробельных символов точно подойдет, про спецсимволы не знаю).
1. После регистрации вы будете автоматически перенаправлены на https://gitlab.manytask.org/ . Если вы проходили эту процедуру ранее для других курсов, и гитлаб вас помнит, то этот шаг автоматически будет пропущен. Если нет, то нужно залогиниться с имеилом и паролем из формы до этого. Нужно разрешить менитаску получать информацию об аккаунте (кликнуть кнопку Authorize), если он попросит.
1. Если все сработало хорошо, то вы окажетесь на главной страничке https://app.manytask.org/spbgu-distsys-2025-fall в своем аккаунте. В будущем, если вы захотите залогиниться на другом устройстве или под другим аккаунтом, авторизация в менитаске происходит через https://gitlab.manytask.org/ при нажатии кнопки login из формы на входной страничке https://app.manytask.org/
1. Наверху страницы https://app.manytask.org/spbgu-distsys-2025-fall вы увидите набор кнопок.
	- Assignments - страница по умолчанию, показывает список заданий, несколько устаревший скриншот с пояснениями в спойлере <details><summary><a>Пояснения интерфейса менитаска</a></summary>[](https://i.imgur.com/FYDgaWj.png)</details>
	- My repo перенаправляет на ваш личный репозиторий данного курса на https://gitlab.manytask.org/
	- My submits открывает страничку ваших посылок. Вначале она пустая, но потом там будут появляться запуски проверок домашек. Знатоки гитлаба узнают в этой страничке список гитлаб пайплайнов. Проверка заданий производится через gitlab CI скрипт. Если вы знакомы с синтаксисом gitlab CI файлов, можете посмотреть .gitlab-ci.yml файл в репозитории. Если не знакомы с синтаксисом, то смотреть файл не нужно, все будет объяснено словами в этой инструкции. Стрелять себе в ногу посредством перезаписи файла бесполезно и не рекомендуется
	- All scores ведет в гугл табличку, в которой лежат оценки всех студентов текущего курса текущего года. Гугл табличка доступна только на чтение, запись происходит после проверки очередной посылки. Вначале идет набор фиксированных колонок, затем идут колонки задачек. При успешном решении задания после окончания пайплайна из My submits оценка автоматически попадет в табличку. Эта табличка не лидерборд в том смысле, что сортировки не происходит, табличка нужна только в качестве единого источника информации для студентов и преподавателей
	- Links содержит разные полезные ссылки, отличается от курса к курсу
1. Далее необходимо настроить доступ к вашему аккаунту https://gitlab.manytask.org/ через ssh ключ. В теории возможно работать через https, но тогда при каждой сдаче задания необходимо будет вводить логин-пароль, что неудобно, а также лишает вас возможности научиться работать с ssh ключами.
	- Если вы уже сдавали что-то в менитаске других курсов и не вводили каждый раз логин-пароль, то скорее всего ssh ключ у вас уже настроен, см. спойлер [как проверить себя](#check-ssh)
	- Устанавливаем git и ssh-keygen или проверяем, что они уже стоят. См. [ниже](#install)
	- Создаем ssh ключ, см. [инструкцию](#ssh).
	- Заходим в Preferences аккаунта в https://gitlab.manytask.org/ , идем в SSH Keys, кликаем Add ssh key, вставляем в большое текстовое поля публичный ключ (важно! Файл с публичным ключом обычно имеет расширение .pub, не перепутайте публичный и приватный ключи), вводим имя и кликаем Add key. Чаще всего, ключ лежит в папке `~/.ssh/`
	1.[Проверяем](#check-ssh) из консоли, что ключ установлен корректно
1. Клонируем ваш личный репозиторий на локальную машинку. Идем по ссылке My repo в менитаске в свой репозиторий, нажимаем Code и копируем строчку для ssh протокола (начинается с `git@gitlab.manytask.org`, в ссылке также должен быть виден ваш менитаск ник). В консоли пишем `git clone <ссылка>`. В результате в той папке, где вы запускали команду git clone, должна появиться новая папка, которая называется как ваш ник в менитаске. Внутри будет лежать репозиторий. Содержимое идентично вашему удаленному репозиторию в https://gitlab.manytask.org/ 
1. Заходим внутрь репозитория `cd <ваш ник в менитаске>` и настраиваем ссылку для обновления заданий. Необходимо находиться внутри папки с локальным репозиторием. Для онбординга пишем `git remote add upstream git@gitlab.manytask.org:spbgu/distsys/public-2025-fall.git`, нужную ссылку для конкретного курса можно получить из менитаска, нажав на любую из плиток задач. Вы попадете в описание задачи в публичном репозитории (что такое публичный репозиторий см. [структуру в файле README](README.md#structure)), там кликаем кнопку Code и копируем ссылку для клонирования через ssh
1. Выполняем задачу
	- Проверяем, что ваш репозиторий содержит актуальные задания, для этого выполняем `git pull upstream main`. Важно, что новые задачи забираются именно из upstream, тк только туда преподаватели заливают новые задачи, ваш удаленный репозиторий автоматически не будет получать новые задания. Скорее всего, у вас уже все загружено и вы увидите в конце вывода Already up to date.
	- Заходим в папку задачи. Например, `cd 1.FirstSteps/HelloWorld/`. Внутри папки задачи обычно есть файлик README.md, где описано что необходимо сделать. Рядом может лежать шаблон решения, как например `hello_world.py`
	- Меняем локально все необходимые файлы, чтобы решить задание. В случае с хелло ворлд добавляем строчку `    return "Hello World!"` в `hello_world.py`, сохраняем файл
	- Часто в папке задачи также лежит набор публичных тестов (конечно, зависит от курса и конкретной задачи), обычно прежде чем отправлять задачу на проверку хорошо удостовериться, что публичные тесты, а также другие локальные проверки (например, линтеры) проходят
	- Чтобы отправить задачу на проверку и получить баллы, нужно запустить gitlab pipeline в вашем репозитории. Это происходит автоматически как только проимходит добавление коммит в вашем удаленном репозитории. Весь процесс выглядит примерно так: добавляем измененные файлы в staging area `git add hello_world.py`, коммитим `git commit -m "Solved hello world!"`, пушим в свой удаленный репозиторий командой `git push origin main`. 
	- Ждем, пока пайплайн успешно завершится. К сожалению, не всегда это происходит мгновенно, тк раннеров ограниченное количество. Найти список пайплайнов можно при переходе по кноке My submits в менитаске. Новые посылки добавляются наверх, если верхняя посылка имеет зеленый значок, значит проверка прошла успешно! Если красный, то что-то пошло не так, нужно кликнуть на красный значок и потом на grade, чтобы увидеть логи. Далее нужно аккуратненько почитать что в логах написано, обращая особое внимание на слово error или красный шрифт. От курса к курсу содержимое логов может значительно отличаться. Проверяться будут только измененные задачи, за один раз может быть проверено несколько задач. Смотреть какие задачи проверяются можно в логах где-то после строчки `Detecting changes by ChangesDetectionType.LAST_COMMIT_CHANGES`, цвет текста может быть настолько неудачным, что текст не видно, но где-то там есть список проверяемых задач :)
	- Если пайплайн упал (красный) и вы поправили ошибку, перепроверка запускается абсолютно таким же образом: нужно снова добавить коммит, задевающий файл в папке задачи, и загрузить его в удаленный репозиторий. В **редких** случаях ошибка может быть связана не с вашим решением, а с неполадками в системе проверки. Тогда пайплайн можно перезапустить через интерфейс гитлаба 
	- Баллы можно смотреть либо в менитаске, либо в гугл табличке. Если сдавалось сразу несколько заданий, то пайплайн может быть зеленым даже при условии что только одно из заданий зачтено, так что скор стоит проверять отдельно

### Преднастройка <a id='pre'/>

<details><summary><a>MacOS</a></summary>

В MacOS не хватает менеджера пакетов для удобно установки чего-либо (по типу apt-get).

Поэтому нужно установить `brew` по [официальной инструкции](https://brew.sh/).
```shell
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```
</details>

<details><summary><a>Windows</a></summary>

В Windows 10 появилась такая фича как WSL: Windows Subsystem for Linux,
с её помощью можно запускать Linux-приложения на Windows.
Мы рекомендуем воспользоваться ею, и в дальнейшем следовать инструкциям,
как будто бы у вас стоит операционная система Linux.

Установить `wsl` нужно по [официальной инструкции](https://docs.microsoft.com/ru-ru/windows/wsl/install-win10)

При выборе операционной системы Linux берите Ubuntu 18.04 или 22.04.

Запустите установленную систему. При входе вы окажетесь в директории `/home/<username>`;
для того, чтобы иметь возможность работать с кодом из самой Windows (например, в PyCharm),
мы рекомендуем размещать директорию с задачами по адресу `"/mnt/c/Users/<username>/My Documents"`,
которая в самой Windows доступна по адресу `C:\Users\<username>\My Documents`.

Перейдите в указанную директорию:
```bash
$ cd "/mnt/c/Users/<username>/My Documents"
```
Следуйте по инструкции дальше

</details>

### Создание ssh-ключа
<a id="ssh"></a>

Если же вы уже были участником других курсов с manytask, то, скорее всего, этот ключ у вас уже есть и 
или можно создать дополнительный или использовать старый (см. пункт "Как проверить себя?")


- Воспользуйтесь `ssh-keygen` (возможно, вам придется поставить `openssh-client`), затем скопируйте **.pub** ключ:
  ```bash
  # Создаем ключ:
  $ ssh-keygen -t ed25519 -f ~/.ssh/manytask_ed25519
  # Обратите внимание, что вы можете не указывать пароль для ключа,
  # чтобы не приходилось его потом вводить на каждое действие c ключом
  # Это стандартная практика, хотя и не очень безопасная
  
  # Выводим содержимое **публичного** ключа в консоль:
  $ cat ~/.ssh/manytask_ed25519.pub
  ...
  # Его надо просто скопировать, как есть, включая подпись - обычно это "ваш-логин@имя-устройства"
  # ВАЖНО! Публичным ключом можно делиться, приватным (то же имя, без .pub на конце) - никогда,
  # иначе злоумышленник сможет представиться вами
  ```

  <details><summary><a>Картинка</a></summary><img src="https://i.imgur.com/FMHgxsL.png" width=800/></details></br>

- Идете на [gitlab.manytask.org](https://gitlab.manytask.org/)

- Жмете на иконку с вашим профилем  -> `Preferences` -> слева жмете на `SSH keys`

- Вставляете ключ в формочку, жмете "Add key"

  <details><summary><a>Картинка</a></summary><img src="https://i.imgur.com/CSPBoGp.png" width=800/></details></br>

- Создайте ssh-config с таким содержимым, чтобы при подключении к `gitlab.manytask.org` использовался ваш новый ключ:
  ```bash
  $ cat ~/.ssh/config
  Host gitlab.manytask.org
      IdentityFile ~/.ssh/manytask_ed25519
  ```
  Создать файл можно с помощью редактора `nano`, если он установлен
  ```bash
  $ nano ~/.ssh/config
  ```
  Затем нужно вставить в файл содержимое и нажать ctrl + O для сохранения и ctrl + X для выхода из редактора.
  
  Либо с помощью команды 
  ```bash
  $ echo $'Host gitlab.manytask.org\n\tIdentityFile ~/.ssh/manytask_ed25519' > ~/.ssh/config
  ```

<a id="check-ssh"></a>
<details><summary><a>Как проверить себя?</a></summary>

Из консоли выполнить:
```bash
$ ssh git@gitlab.manytask.org
# Вывод должен быть примерно таким:

Welcome to GitLab, @hiverus!

```
</details>

#### Клонирование и настройка репозитория

```bash
# Переходим в директорию, где вы хотите разместить репозиторий с задачами,
# обычно это домашняя директория - `/home/<username>` или /Users/<username>, она же обозначается как `~` (это тильда, если что)
$ cd <твоя выбранная директория>

# Клонируем свой репозиторий с gitlab.manytask.org (создается автоматически при регистрации)
# Имя репозитория доступно по ссылке "MY REPO" на менитаске
$ git clone git@gitlab.manytask.org:python/students-fall-2024/<твой репозиторий>

# Переходим в директорию склонированного репозитория
$ cd <твой репозиторий>

# Настраиваем гит так, чтобы он знал нас "в лицо"
$ git config --local user.name "<твой логин с manytask.org>"
```
